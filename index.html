HTML

  • HTML stands for Hypertext Markup Language
  • The head element contains information about the webpage.
  • The body element represents visible content the user can see.
  • The p(aragraph) element represents a block of text in a paragraph.
  • The h(1 through 6) element represents headers that are largest to smallest from 1 to 6.
  • The img element is used to place an image file and requires a src attribute describing where the image is stored and an alt attribute that shows alternative text for when the image does not load.
  • The footer element is just to creater a footer at the bottom of the page
  • Save with .htm or .html at end of filename.

CSS

  • CSS stands for Cascading Style Sheets
  • Creating a style for a given element usually consists of a selector, and properties with corresponding values.
  • CSS sheets must be linked to the html doc in the head section using a link wtih rel and href properties.
  • Margin indicates how much space we want around the outside of an element.
  • padding indicates how much space we want around the content inside an element.
  • Typing a period (.) before a selector designates it as a class selector.
  • The names of css selectors need to match class or id names in the html doc in order to apply changes to specific elements/sections.
  • CSS docs use .css at end of filename

GIT

  • git status: this checks what branch currently in.
  • git checkout -b 'branch-name' creates a new branch and switches to it.
  • After making changes to a doc saved in git repository, run: git add -A then run git commit -m "message describing fix/changes"
  • git pull origin main: sync local env with main branch of github repo.
  • git push origin 'branch name': send local changes to github repository
  • Always work on a feature branch (a branch other than main).

JavaScript

  • A JavaScript file is linked to html doc with script element at the end of the body parent element with src attribute
  • JavaScript docs use .js in filename.
  • All strings are contained within double quotes ""
  • var declares a variable, which is a placeholder which lets us store data in our program.
  • comments start with /* and end with */
  • a function name is always followed by parentheses
  • Functions are useful blocks of code that perform certain tasks and can be called with the function name after they have been defined
  • use camelCase for names involving more than one word.
  • Control flow describes the order that code is read by the computer to run a script.
